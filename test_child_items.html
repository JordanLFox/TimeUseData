Child Items Feature Implementation

This document provides a summary of the child items feature implementation for the timeline activity diary app.

JSON Schema Updates

The JSON schema has been updated to support child items for activities:

{
  "name": "washing, dressing",
  "examples": "",
  "color": "#a8e3f0",
  "childItems": [
    {
      "name": "Washing and dressing",
      "color": "#a8e3f0"
    },
    {
      "name": "Other personal care",
      "color": "#a8d3f0"
    }
  ]
}

Example Activities with Child Items

  • sleeping - Nighttime sleep, Nap/Rest
  • washing, dressing - Washing and dressing, Other personal care
  • eating, drinking - Breakfast, Lunch, Dinner, Snack

Feature Implementation

The following updates were made to implement the child items feature:

  1. JSON Schema Update: Added childItems array to activity objects
  2. Visual Indicator: Activities with child items show a dropdown icon
  3. Child Items Modal: Modal displays when parent activity is selected
  4. Data Storage: Child items store reference to parent activity

Note: When a child item is selected, the parent activity name is stored in the parentName property in the activity data, making it easy to identify child activities and their parent activities when processing the data.

How It Works

  1. User clicks on an activity with child items (indicated by dropdown icon)
  2. A modal appears showing all child items
  3. User selects a specific child item
  4. The selected child item is placed on the timeline
  5. Both the child item name and parent name are stored in the activity data

CSS Styling

Custom CSS was added to style the child items modal and to visually indicate activities with child items:

.activity-button.has-child-items::after {
    content: '\25BC'; /* Downward-pointing triangle */
    position: absolute;
    right: 10px;
    top: 50%;
    transform: translateY(-50%);
    font-size: 10px;
    color: rgba(0, 0, 0, 0.5);
}
